Cuando Gibran pide "una propuesta nueva", "redesign", "otra versión bien distinta" para un website/landing/mockup SIN proporcionar referencia visual o brief específico, el output suele descartarse después de 1-2 iteraciones.

**Why:** 2026-05-02 — intento de spekgen.com/v1. Iteración 1 (dark editorial Instrument Serif + Lenis + GSAP + custom cursor) descartada por hero invisible, cursor "horrible", scroll torpe. Iteración 2 (light SPEKGEN.OS + Geist + dashboard modular + package.json) descartada con "no me gusta nada". Resultado: page borrada de Shopify, redirects eliminados, archivo local eliminado. Trabajo desperdiciado en dos direcciones bold pero a ciegas.

**How to apply:** Antes de escribir HTML/CSS para un redesign, pedir UNO de:
- 2-3 screenshots de sites que le gustan (Awwwards, Linear, etc.)
- Cliente / aesthetic reference ("quiero que se vea como X")
- Mood board específico
- Lista de qué NO quiere del diseño actual

Si Gibran insiste en "tú propón algo bien distinto" sin dar reference, advertir explícitamente: "sin referencia voy a divagar; ¿prefieres que primero te muestre 3-4 thumbnails / direcciones rápidas para elegir, antes de construir el archivo completo?". Default a thumbnail-first cuando no hay reference clara.

Aplica a: spekgen.com redesigns, mockups de prospectos sin design DNA en `_design_intel/reports/`, landings nuevas, cualquier cambio visual mayor.
