HTML/CSS slides for social media posts must NOT be generic/empty. Standards:

- **S1 (Hook) SIEMPRE con Gemini** — nunca solo HTML/CSS con mockup. Necesita imagen generada con elementos decorativos, contexto visual, estética premium.
- **Nunca usar solo el mockup** — se ve feo solo. Siempre compositar con elementos: flores, texturas, fondos elaborados, lifestyle.
- **Colores ricos, no genéricos** — adaptar paleta al territorio del producto (púrpura/dorado para hormonal, azul noche para sueño, etc.). No usar solo mint green para todo.
- **Tipografías ligeras y elegantes** — Playfair Display bold es demasiado pesado. Usar fonts más atractivas y menos densas.
- **Elementos complementarios** — formas orgánicas, gradientes suaves, texturas, decoraciones. No slides vacíos con solo texto y cards.
- **Densidad visual** — cada slide debe sentirse completo visualmente, no un wireframe.

**Why:** El primer intento con T1 puro (GR-004 abril 2026) fue rechazado por verse vacío, genérico y feo. El mockup solo sin contexto visual no funciona.

**How to apply:** Para cada post, S1 SIEMPRE genera con Gemini. Slides internas pueden ser CSS pero con diseño elaborado. Adaptar colores al producto/territorio. Usar tipografías elegantes.
