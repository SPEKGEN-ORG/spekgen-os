# No metas features no solicitadas en mockups/propuestas de prospects

**Regla:** Cuando construyas un mockup o propuesta para un prospect basándote en el transcript de la llamada con Gibran, **apégate ESTRICTO a lo que el prospecto pidió**. Si crees que algo extra aporta valor (cuenta B2B, tabla comparativa, número específico de SKUs, promesa de marca), **menciónalo a Gibran como sugerencia consultiva ANTES de meterlo al mockup**.

**Why:**
- Gibran cachó que el mockup de Ferre24 (2026-05-02) tenía toda una sección "Cuenta B2B Empresarial" + email b2b + nav link + tab "Precio B2B" + catálogo "B2B Empresarial" en biblioteca + USP strip bullet "Cuenta B2B" — y NADA de eso estaba en el transcript de Sergio. Yo lo metí asumiendo que tenía sentido para ferretería express GDL (contratistas/mantenimiento corporativo son cliente natural)
- Cuando Gibran le presente el mockup a Sergio y Sergio diga "no, yo no quiero esto", queda mal: parece que no escuchamos
- Otros añadidos sospechosos en ese mismo mockup: tabla comparativa Ferre24 vs Home Depot/MC/Truper, "2,500 SKUs curados" como número específico, cobertura definida en 5 municipios (GDL/Zapopan/Tlaquepaque/Tonalá/Tlajomulco), tesis "express en 24h" formalizada como promesa de marca
- Tuvimos que hacer cleanup en sesión siguiente — desperdicio de tokens y contexto

**How to apply:**
- Antes de construir mockup/propuesta de prospect: re-leer transcript del call y listar **textualmente** qué pidió el prospect
- Cualquier feature que NO esté en esa lista: marcar como "sugerencia consultiva" y CONSULTAR con Gibran si la incluimos antes de codear
- Si Gibran dice "métela", incluirla con label visible "[ Sugerencia SPEKGEN — opcional ]" para que Sergio sepa que no lo pidió pero se lo proponemos
- Aplica a: B2B / cuenta empresarial / crédito / descuentos volumen / login / dashboard / comparativas / números específicos / promesas formales de marca / coberturas geográficas detalladas / cualquier "feature avanzado" no mencionado por el prospect
- En propuesta económica está OK incluir add-ons opcionales — pero el mockup base se apega al brief estricto

**Excepciones (sí aplica meter sin preguntar):**
- Elementos básicos que cualquier sitio web tiene (header, footer, contacto, social links si tiene redes)
- Marca de mockup SPEKGEN (ribbon, badge, watermark) — siempre va
- Placeholder copy/imagery cuando no hay assets reales
