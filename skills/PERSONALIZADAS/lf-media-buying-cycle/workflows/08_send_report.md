# Workflow 08 — Send report (cada 3 días)

> Trigger: cron `0 15 */3 * *` (9 AM MX cada 3 días) — Japan-proof.
> Output: email a `gibran.alonzo0506@gmail.com` con resumen ejecutivo + decisiones autónomas.

---

## Objetivo

Mantener a Gibran en el loop sin necesitar que él revise activamente. Email cada 3 días con: estado cuenta, top/bottom performers, decisiones autónomas tomadas, pregunta puntual si necesita input.

**Diseño:** legible en mobile en <30 segundos. Si Gibran está en Japón sin WiFi, este email es su único canal — debe ser autosuficiente.

## Pre-flight

Cargar:
1. `_QUICK_REFERENCE.md`
2. `LOGS/LF/CALIBRATION.md`
3. `LOGS/LF/DECISIONS_LOG.md` — desde último report (3d back)
4. `LOGS/LF/BATCH_LOG.md` — ads activos
5. `LOGS/LF/MONTH_PLAN_MAY26.md` — vs progreso

## Procedimiento

### Paso 1 — Pull cuenta-nivel últimos 3d / 7d / 30d

```bash
python3 scripts/pull_meta_metrics.py --account-summary --days 3
python3 scripts/pull_meta_metrics.py --account-summary --days 7
python3 scripts/pull_meta_metrics.py --account-summary --days 30
```

### Paso 2 — Pull top 3 + bottom 3 ads activos (últimos 7d)

```bash
python3 scripts/pull_meta_metrics.py --all-active --days 7
```

Sortear por ROAS desc → top 3. Sortear ascendente → bottom 3.

### Paso 3 — Listar decisiones autónomas tomadas desde último report

Leer `DECISIONS_LOG.md` filtrando por timestamps últimos 3 días. Resumir cada D-NNN en 1 línea.

### Paso 4 — Detectar si requiere input de Gibran

Triggers de pregunta puntual:
- Cuenta sostenida ROAS <1.0× 7d → "¿pause global?" (kill switch — no autorizado autónomo)
- Stock crítico de winner cae <50 unidades → "¿re-stock o pivote?"
- Budget alcanzando ceiling $333/d ceiling 5+ días seguidos → "¿subimos a $400?"
- Gemini billing fail recurrente → "¿retry con otra cuenta?"
- Gap de PDA en VoC mining (no hay material para próximo brief) → "¿extender ventana de mining?"

Si NO hay trigger: pregunta es "todo OK, sigo según plan". Si sí: pregunta puntual.

### Paso 6 — Enviar via Gmail SMTP

Usar `SPEKGEN_GMAIL_APP_PASSWORD` (env var). Enviar desde `gibran.alonzo0506@gmail.com` o cuenta SPEKGEN dedicada. Subject: `LF Media Buying — {fecha} — {emoji status}` donde emoji = 🟢 OK / 🟡 alerta / 🔴 input requerido.

```python
import smtplib

msg["From"] = "..."
msg["To"] = "gibran.alonzo0506@gmail.com"
msg["Subject"] = f"LF Media Buying — {date} — {status_emoji}"
msg.set_content("HTML version below")
msg.add_alternative(html_content, subtype="html")
with smtplib.SMTP("smtp.gmail.com", 587) as s:
    s.starttls()
    s.login(user, password)
    s.send_message(msg)
```

### Paso 7 — Log

Append a `LOGS/LF/REPORTS/report_{YYYY-MM-DD}.html` (copia local del email enviado).

## Edge cases

- **Email send fail** (SMTP timeout, etc.): retry 3× backoff. Si sigue → fallback notification a Discord webhook (futuro). Por ahora log local + bandera URGENT en próximo workflow.
- **Cuenta sin actividad** (no spend 7d): aún enviar report con cero data + bandera "WHY NO SPEND" para investigar.

## Output esperado

1. Email enviado a `gibran.alonzo0506@gmail.com`
2. `LOGS/LF/REPORTS/report_YYYY-MM-DD.html` archivo local
3. Append a `DECISIONS_LOG.md`: `Report enviado HH:MM, status emoji X`
